Constraint check

What changed and what didn't, for review against the base brief + amendment 1.

  • Unchanged (hard constraints): canonical status→colour map; circle=step/diamond=gate/dashed=start-gate; team-tint scoped to the avatar, never the node; pulse reserved for the active node (the only attention-seeking motion, per the base brief's motion rules).
  • New — node rendering: the active node's centre dot (the "ictus") reuses the identity's own baton/ictus vocabulary rather than inventing a new motif; the gate diamond gets a static (non-animated) brass halo matching the base brief's "its scarcity is the product's personality"; blocked is now a dashed hollow ring (still hollow neutral, still explicitly labelled below it) so it reads distinctly from waiting without touching colour.
  • New — the line: three states (settled / frontier / future) instead of one flat hairline, all derived from tokens the palette already owns (green-tinted thread, active-or-gate-tinted frontier, neutral dashed future) — a richer drawing of existing semantics, not a new channel.
  • New — row wash: the active/needs-you row gets the same actionable-tint treatment already shipped for gate cards and stat cells (--gate-panel) — generalised to --active-panel for "this step is live right now", not a new idea.
  • New — rhythm: node, avatar, and title share one fixed centerline (part 1's fix, carried forward); the avatar gets a 2px paper-coloured ring so it reads as its own disc against the new row washes.
  • Not yet decided: whether the start-gate node actually gets wired into scoreNodes() — flagged above, a data-layer question for the Conductor, not a rendering one.
