02Components

One component per idea. Every variant has a documented meaning.

Buttons — three variants, hierarchy enforced
Rule: one primary (dark) per view. Destructive actions always get the danger variant — previously Reject was styled identically to the neutral Re-scope. Loading behaviour is specified in §05.
Chips — metadata only, never status
product-brief claude-sonnet-5 pace · auto GITHUB_TOKEN LINEAR_API_KEY agent eval
Rule: filled neutral chip = metadata; dashed chip = env/secret reference; uppercase pill = kind tag (top-right of every registry card, no exceptions). Dark chips are gone.
Status — one taxonomy, one rendering, with legend
approved at gate exhausted running superseded offline
Rule: dot + uppercase label, tinted background, always the same geometry. exhausted is red (blocks the loop); at gate is amber (waits on you); green means healthy / approved / online, everywhere. running carries the only animated dot in the system — a slow pulse meaning "live right now" (see §05).

05Loading & live states

The state indicator must match the duration of the wait. Three tiers cover every work-unit action on studio, project and work-unit pages. The demos below are live — click the buttons.

The three tiers
tier 1 · button0–1s
Decision writes (approve / re-scope / reject). Pressed state at 0ms, spinner after 150ms, progressive label (Approving…), minimum 500ms once shown. The whole action group disables; the spinner rides only on the clicked button.
tier 2 · card1–10s
Resolution and refetch. The card never blanks: badge swaps in 200ms, meta line updates, section counts and stat band tick in the same beat. If the card moves sections (Needs you → Running now), it animates over 240ms.
tier 3 · loop10s+
Long-running loops. A spinner would be a lie — the card flips to running with a live strip: pulsing dot, round n/m, elapsed and tokens ticking. Full expression lives on the run view (S9).
Button loading anatomy (static)
One in-flight action per card: siblings disable, the clicked button shows spinner + progressive label. Width is preserved by keeping the label — no layout shift in the footer.
